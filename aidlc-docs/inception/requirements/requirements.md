# 테이블오더 서비스 - 요구사항 문서

## Intent Analysis Summary
- **User Request**: 테이블오더 서비스 신규 구축
- **Request Type**: New Project (Greenfield)
- **Scope**: System-wide (고객 UI, 관리자 UI, 백엔드 서버, 데이터베이스)
- **Complexity**: Complex (실시간 통신, 세션 관리, 인증, 다수 기능)

---

## 기술 스택 결정사항

| 영역 | 기술 | 비고 |
|------|------|------|
| 프론트엔드 | React (Vite + TypeScript) | SPA, 단일 앱에서 고객/관리자 라우팅 분리 |
| 백엔드 | Java + Spring Boot | REST API + SSE |
| 데이터베이스 | H2 Database | 파일 기반 모드, Spring Boot 내장 지원 |

| 배포 환경 | 로컬 실행 | 클라우드 배포 없음 |

## Functional Requirements

### FR-01: 테이블 태블릿 연결 및 세션 관리
- **테이블 등록**: 관리자가 관리자 UI에서 테이블 등록 (테이블 번호 + 비밀번호 설정) → 시스템이 고유 액세스 토큰 생성
- **태블릿 초기 설정**: 관리자가 태블릿에서 1회 토큰 입력 → localStorage에 저장
- **고객 접근**: 고객은 로그인 절차 없이 태블릿을 켜면 저장된 토큰으로 자동 인증되어 바로 메뉴 화면 표시
- **인증 세션**: 토큰 기반, 만료 없음 (관리자가 테이블 재등록 시에만 변경)
- **테이블 세션과 인증은 독립적**: 테이블 세션은 관리자가 "이용 완료" 시 종료, 토큰 인증은 계속 유지

### FR-02: 메뉴 조회 및 탐색
- 메뉴 화면이 기본 화면
- 카테고리별 메뉴 분류 및 표시
- 메뉴 상세 정보: 메뉴명, 가격, 설명, 이미지
- 카테고리 간 빠른 이동
- 카드 형태 레이아웃, 터치 친화적 (최소 44x44px)

### FR-03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절
- 총 금액 실시간 계산
- 장바구니 비우기
- 클라이언트 측 로컬 저장 (페이지 새로고침 시 유지)
- 서버 전송은 주문 확정 시에만

### FR-04: 주문 생성
- 주문 내역 최종 확인 후 주문 확정
- 주문 성공: 주문 번호 표시 → 5초 후 메뉴 화면 자동 리다이렉트 → 장바구니 자동 비우기
- 주문 실패: 에러 메시지 표시, 장바구니 유지
- 주문 정보: 매장 ID, 테이블 ID, 메뉴 목록(메뉴명, 수량, 단가), 총 금액, 세션 ID

### FR-05: 주문 내역 조회
- 현재 테이블 세션 주문만 표시
- 주문 시간 순 정렬
- 주문별: 주문 번호, 시각, 메뉴/수량, 금액, 상태(대기중/준비중/완료)
- SSE로 주문 상태 실시간 업데이트 (MVP 포함)
- 매장 이용 완료 처리된 주문 제외

### FR-06: 매장 관리자 인증
- 매장 식별자 + 사용자명 + 비밀번호 로그인
- 회원가입 기능 포함 (다수 관리자 지원)
- JWT 토큰 기반 인증
- 16시간 세션 유지, 브라우저 새로고침 시 유지
- 비밀번호 bcrypt 해싱
- 로그인 시도 제한

### FR-07: 실시간 주문 모니터링
- SSE 기반 실시간 주문 목록 업데이트
- 그리드/대시보드 레이아웃: 테이블별 카드 형태
- 각 테이블 카드: 총 주문액, 최신 주문 미리보기
- 주문 카드 클릭 시 전체 메뉴 목록 상세 보기
- 주문 상태 변경 (대기중 → 준비중 → 완료)
- 신규 주문 시각적 강조
- 2초 이내 주문 표시

### FR-08: 테이블 관리
- 테이블 태블릿 초기 설정 (테이블 번호, 비밀번호)
- 주문 삭제 (확인 팝업 → 즉시 삭제 → 총 주문액 재계산)
- 테이블 세션 종료 (이용 완료): 주문 내역 과거 이력 이동, 현재 주문/총액 리셋
- 과거 주문 내역 조회 (시간 역순, 날짜 필터링)

### FR-09: 메뉴 관리 (CRUD)
- 메뉴 조회 (카테고리별)
- 메뉴 등록: 메뉴명, 가격, 설명, 카테고리, 이미지 (로컬 파일 업로드)
- 메뉴 수정, 삭제
- 메뉴 노출 순서 조정
- 필수 필드 검증, 가격 범위 검증

---

## Non-Functional Requirements

### NFR-01: 성능
- SSE 기반 실시간 통신 (주문 2초 이내 표시)
- 클라이언트 측 장바구니 관리 (서버 부하 최소화)

### NFR-02: 사용성
- 터치 친화적 UI (최소 44x44px 버튼)
- 직관적인 카드 형태 레이아웃
- 명확한 시각적 계층 구조

### NFR-03: 데이터 관리
- H2 파일 기반 모드 (데이터 영속성)
- 세션 ID 기반 주문 그룹화
- 주문 이력 보존 (OrderHistory)

### NFR-04: 이미지 관리
- 로컬 파일 시스템 저장
- 이미지 URL은 서버 경로로 제공

## 제외 기능 (구현하지 않음)
- 결제 처리 (카드, 현금, PG사 연동)
- OAuth/SNS 로그인, 2FA
- 이미지 리사이징/최적화
- 푸시/SMS/이메일 알림
- 주방 기능 (주방 전달, 재고 관리)
- 데이터 분석, 매출 리포트
- 직원 관리/권한 설정, 예약, 리뷰, 다국어
- 외부 연동 (배달, POS, 소셜미디어)

---

## 용어 정의
- **테이블 세션**: 고객이 첫 주문 시작부터 이용 완료 처리까지의 기간. 종료 후 새 고객의 첫 주문 시 새 세션 시작.
- **MVP**: Minimum Viable Product (최소 기능 제품)
