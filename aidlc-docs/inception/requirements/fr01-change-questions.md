# FR-01 변경 명확화 질문

고객이 태블릿에서 로그인 없이 바로 주문하는 구조로 변경합니다.

---

## Question 1
태블릿이 특정 테이블과 연결되는 방식은?

### Option A: URL 기반 연결
- 관리자가 테이블 등록 시, 시스템이 테이블별 고유 URL을 생성 (예: `http://localhost:3000/table/3`)
- 관리자가 태블릿 브라우저에 해당 URL을 북마크/홈화면에 설정
- 고객이 태블릿을 켜면 바로 해당 테이블의 메뉴 화면이 표시됨
- **장점**: 설정이 매우 간단, 별도 인증 로직 불필요, URL 자체가 테이블 식별자
- **단점**: URL을 아는 사람은 누구나 접근 가능 (매장 내부 네트워크라면 문제 없음)
- **구현**: 프론트엔드 라우팅만으로 처리 (`/table/:tableNumber`), 백엔드에서 테이블 번호로 메뉴/주문 조회

### Option B: 토큰 기반 연결
- 관리자가 테이블 등록 시, 시스템이 고유 액세스 토큰을 생성
- 관리자가 태블릿에서 1회 토큰을 입력하면, 토큰이 localStorage에 저장됨
- 이후 고객이 태블릿을 켜면 저장된 토큰으로 자동 인증되어 메뉴 화면 표시
- **장점**: 토큰 없이는 접근 불가, 보안이 더 강함
- **단점**: 초기 1회 토큰 입력 과정 필요, 토큰 관리 로직 추가 구현 필요
- **구현**: 백엔드에서 토큰 검증 API 필요, 프론트엔드에서 토큰 저장/전송 로직 필요
