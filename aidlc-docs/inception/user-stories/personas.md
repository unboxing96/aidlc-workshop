# User Personas

## Persona 1: 고객 (Customer)

| 항목 | 내용 |
|------|------|
| **이름** | 김고객 |
| **역할** | 매장 방문 고객 |
| **설명** | 매장 테이블에 앉아 태블릿으로 음식을 주문하는 일반 고객 |
| **목표** | 빠르고 편리하게 메뉴를 탐색하고 주문하기 |
| **동기** | 직원 호출 없이 원하는 시점에 즉시 주문, 주문 상태 실시간 확인 |
| **페인포인트** | 복잡한 UI, 느린 응답, 주문 상태 불확실성 |
| **기술 수준** | 다양 (스마트폰 사용 가능 수준이면 충분) |
| **사용 환경** | 매장 내 테이블 태블릿 (브라우저 기반) |

---

## Persona 2: 관리자 (Admin)

| 항목 | 내용 |
|------|------|
| **이름** | 박관리자 |
| **역할** | 매장 운영 관리자 |
| **설명** | 매장의 주문을 실시간으로 모니터링하고, 메뉴/테이블을 관리하는 운영자 |
| **목표** | 효율적인 매장 운영, 실시간 주문 파악, 메뉴/테이블 관리 |
| **동기** | 인건비 절감, 주문 누락 방지, 운영 효율성 극대화 |
| **페인포인트** | 주문 누락, 테이블 상태 파악 어려움, 수동 메뉴 관리 번거로움 |
| **기술 수준** | 기본적인 웹 애플리케이션 사용 가능 |
| **사용 환경** | 매장 내 PC 또는 태블릿 브라우저 |
