# 고객 (Customer) 스토리

## Epic: 테이블 태블릿 접근 (FR-01)

### US-C01: 태블릿에서 바로 메뉴 접근
**As a** 고객,
**I want to** 태블릿을 켜면 별도 로그인 없이 바로 메뉴 화면을 보고 싶다,
**So that** 즉시 주문을 시작할 수 있다.

**Acceptance Criteria:**
- Given 관리자가 초기 설정을 완료한 태블릿에서, When 고객이 브라우저를 열면, Then 저장된 토큰으로 자동 인증되어 메뉴 화면이 바로 표시된다.
- Given 토큰이 설정되지 않은 태블릿에서, When 브라우저를 열면, Then 토큰 입력 화면이 표시된다 (관리자용 초기 설정).

## Epic: 메뉴 조회 (FR-02)

### US-C03: 카테고리별 메뉴 조회
**As a** 고객,
**I want to** 카테고리별로 분류된 메뉴를 볼 수 있길 원한다,
**So that** 원하는 종류의 메뉴를 빠르게 찾을 수 있다.

**Acceptance Criteria:**
- Given 메뉴 화면에 접근했을 때, When 화면이 로드되면, Then 카테고리 목록과 해당 카테고리의 메뉴가 카드 형태로 표시된다.
- Given 메뉴 화면에서, When 다른 카테고리를 선택하면, Then 해당 카테고리의 메뉴 목록으로 빠르게 이동한다.

### US-C04: 메뉴 상세 정보 확인
**As a** 고객,
**I want to** 메뉴의 상세 정보(이름, 가격, 설명, 이미지)를 확인하고 싶다,
**So that** 주문 전에 메뉴에 대해 충분히 파악할 수 있다.

**Acceptance Criteria:**
- Given 메뉴 카드가 표시될 때, When 카드를 확인하면, Then 메뉴명, 가격, 설명, 이미지가 모두 표시된다.
- Given 메뉴 이미지가 없을 때, When 메뉴 카드가 표시되면, Then 기본 플레이스홀더 이미지가 표시된다.

## Epic: 장바구니 관리 (FR-03)

### US-C05: 장바구니에 메뉴 추가
**As a** 고객,
**I want to** 원하는 메뉴를 장바구니에 추가하고 싶다,
**So that** 여러 메뉴를 모아서 한번에 주문할 수 있다.

**Acceptance Criteria:**
- Given 메뉴 화면에서, When 메뉴 추가 버튼을 터치하면, Then 해당 메뉴가 장바구니에 추가되고 장바구니 수량이 업데이트된다.
- Given 이미 장바구니에 있는 메뉴를, When 다시 추가하면, Then 해당 메뉴의 수량이 1 증가한다.

### US-C06: 장바구니 수량 조절 및 삭제
**As a** 고객,
**I want to** 장바구니에서 메뉴 수량을 조절하거나 삭제하고 싶다,
**So that** 주문 전에 원하는 대로 수정할 수 있다.

**Acceptance Criteria:**
- Given 장바구니에 메뉴가 있을 때, When 수량 증가/감소 버튼을 터치하면, Then 수량이 변경되고 총 금액이 실시간으로 재계산된다.
- Given 장바구니에서 메뉴 수량을 0으로 줄이면, When 수량이 0이 되면, Then 해당 메뉴가 장바구니에서 삭제된다.
- Given 장바구니에 메뉴가 있을 때, When 삭제 버튼을 터치하면, Then 해당 메뉴가 장바구니에서 제거된다.

### US-C07: 장바구니 데이터 유지
**As a** 고객,
**I want to** 페이지를 새로고침해도 장바구니 내용이 유지되길 원한다,
**So that** 실수로 새로고침해도 담아둔 메뉴를 잃지 않는다.

**Acceptance Criteria:**
- Given 장바구니에 메뉴가 담겨있을 때, When 브라우저를 새로고침하면, Then 장바구니 내용이 그대로 유지된다.

### US-C08: 장바구니 비우기
**As a** 고객,
**I want to** 장바구니를 한번에 비울 수 있길 원한다,
**So that** 처음부터 다시 메뉴를 선택할 수 있다.

**Acceptance Criteria:**
- Given 장바구니에 메뉴가 있을 때, When 장바구니 비우기 버튼을 터치하면, Then 모든 메뉴가 삭제되고 총 금액이 0원으로 표시된다.

---

## Epic: 주문 생성 (FR-04)

### US-C09: 주문 확정
**As a** 고객,
**I want to** 장바구니의 메뉴를 확인하고 주문을 확정하고 싶다,
**So that** 원하는 메뉴가 주방에 전달된다.

**Acceptance Criteria:**
- Given 장바구니에 메뉴가 있을 때, When 주문 확정 버튼을 터치하면, Then 주문 내역 최종 확인 화면이 표시된다.
- Given 주문 확인 화면에서, When 최종 확정을 하면, Then 주문이 서버로 전송되고 주문 번호가 표시된다.
- Given 주문이 성공했을 때, When 5초가 경과하면, Then 메뉴 화면으로 자동 리다이렉트되고 장바구니가 비워진다.
- Given 주문 전송이 실패했을 때, When 에러가 발생하면, Then 에러 메시지가 표시되고 장바구니는 유지된다.

---

## Epic: 주문 내역 조회 (FR-05)

### US-C10: 현재 세션 주문 내역 조회
**As a** 고객,
**I want to** 현재 테이블 세션의 주문 내역을 확인하고 싶다,
**So that** 지금까지 주문한 내용과 금액을 파악할 수 있다.

**Acceptance Criteria:**
- Given 주문 내역 화면에 접근했을 때, When 화면이 로드되면, Then 현재 테이블 세션의 주문만 시간 순으로 표시된다.
- Given 주문 내역이 표시될 때, When 각 주문을 확인하면, Then 주문 번호, 시각, 메뉴/수량, 금액, 상태가 표시된다.
- Given 이전 세션(이용 완료 처리된)의 주문은, When 주문 내역 화면에서, Then 표시되지 않는다.

### US-C11: 주문 상태 실시간 업데이트
**As a** 고객,
**I want to** 주문 상태가 실시간으로 업데이트되길 원한다,
**So that** 주문이 어떤 단계에 있는지 바로 알 수 있다.

**Acceptance Criteria:**
- Given 주문 내역 화면을 보고 있을 때, When 관리자가 주문 상태를 변경하면, Then SSE를 통해 상태가 실시간으로 업데이트된다 (대기중→준비중→완료).

# 관리자 (Admin) 스토리

## Epic: 매장 관리자 인증 (FR-06)

### US-A01: 관리자 회원가입
**As a** 관리자,
**I want to** 매장 관리 시스템에 회원가입하고 싶다,
**So that** 매장 운영을 위한 관리자 계정을 생성할 수 있다.

**Acceptance Criteria:**
- Given 회원가입 화면에서, When 사용자명과 비밀번호를 입력하고 가입 버튼을 누르면, Then 관리자 계정이 생성되고 로그인 화면으로 이동한다.
- Given 이미 존재하는 사용자명으로, When 회원가입을 시도하면, Then 중복 에러 메시지가 표시된다.

### US-A02: 관리자 로그인
**As a** 관리자,
**I want to** 사용자명과 비밀번호로 로그인하고 싶다,
**So that** 매장 관리 기능에 접근할 수 있다.

**Acceptance Criteria:**
- Given 로그인 화면에서, When 올바른 사용자명과 비밀번호를 입력하면, Then JWT 토큰이 발급되고 관리자 대시보드로 이동한다.
- Given 로그인 후, When 브라우저를 새로고침하면, Then 세션이 유지되어 재로그인 없이 사용 가능하다.
- Given 로그인 후 16시간이 경과하면, When 다음 요청 시, Then 자동 로그아웃되고 로그인 화면으로 이동한다.
- Given 잘못된 credentials로, When 반복 로그인 시도하면, Then 로그인 시도가 제한된다.

## Epic: 실시간 주문 모니터링 (FR-07)

### US-A03: 테이블별 주문 대시보드 조회
**As a** 관리자,
**I want to** 테이블별로 현재 주문 상황을 한눈에 보고 싶다,
**So that** 매장 전체 주문 현황을 빠르게 파악할 수 있다.

**Acceptance Criteria:**
- Given 관리자 대시보드에 접근했을 때, When 화면이 로드되면, Then 테이블별 카드가 그리드 형태로 표시되고 각 카드에 총 주문액과 최신 주문 미리보기가 표시된다.
- Given 테이블 카드를, When 클릭하면, Then 해당 테이블의 전체 주문 메뉴 목록 상세가 표시된다.

### US-A04: 실시간 신규 주문 수신
**As a** 관리자,
**I want to** 새로운 주문이 들어오면 실시간으로 알고 싶다,
**So that** 주문을 놓치지 않고 즉시 처리할 수 있다.

**Acceptance Criteria:**
- Given 대시보드를 보고 있을 때, When 고객이 새 주문을 생성하면, Then SSE를 통해 2초 이내에 해당 테이블 카드에 주문이 표시된다.
- Given 신규 주문이 도착했을 때, When 카드가 업데이트되면, Then 시각적 강조(색상 변경, 애니메이션)로 신규 주문임을 알린다.

### US-A05: 주문 상태 변경
**As a** 관리자,
**I want to** 주문 상태를 변경하고 싶다 (대기중→준비중→완료),
**So that** 주문 처리 진행 상황을 추적하고 고객에게 알릴 수 있다.

**Acceptance Criteria:**
- Given 주문 상세에서, When 상태 변경 버튼을 클릭하면, Then 주문 상태가 다음 단계로 변경된다 (대기중→준비중→완료).
- Given 주문 상태가 변경되면, When 변경이 저장되면, Then 고객 화면에도 SSE를 통해 실시간 반영된다.

## Epic: 테이블 관리 (FR-08)

### US-A06: 테이블 등록
**As a** 관리자,
**I want to** 새로운 테이블을 등록하고 싶다,
**So that** 태블릿에서 해당 테이블로 접근할 수 있다.

**Acceptance Criteria:**
- Given 테이블 관리 화면에서, When 테이블 번호를 입력하고 등록하면, Then 새 테이블이 생성되고 고유 액세스 토큰이 발급되어 표시된다.
- Given 이미 존재하는 테이블 번호로, When 등록을 시도하면, Then 중복 에러 메시지가 표시된다.
- Given 발급된 토큰을, When 태블릿에서 1회 입력하면, Then 토큰이 localStorage에 저장되고 이후 고객이 바로 메뉴 화면에 접근할 수 있다.

### US-A07: 주문 삭제
**As a** 관리자,
**I want to** 특정 주문을 삭제하고 싶다,
**So that** 잘못된 주문을 직권으로 수정할 수 있다.

**Acceptance Criteria:**
- Given 테이블 주문 상세에서, When 주문 삭제 버튼을 클릭하면, Then 확인 팝업이 표시된다.
- Given 확인 팝업에서, When 삭제를 확정하면, Then 주문이 즉시 삭제되고 테이블 총 주문액이 재계산된다.
- Given 삭제가 완료되면, When 결과가 반환되면, Then 성공/실패 피드백이 표시된다.

### US-A08: 테이블 이용 완료 처리
**As a** 관리자,
**I want to** 테이블 이용 완료를 처리하고 싶다,
**So that** 새 고객이 이전 주문 없이 깨끗한 상태에서 시작할 수 있다.

**Acceptance Criteria:**
- Given 테이블 카드에서, When 이용 완료 버튼을 클릭하면, Then 확인 팝업이 표시된다.
- Given 확인 팝업에서, When 이용 완료를 확정하면, Then 현재 세션의 주문 내역이 과거 이력으로 이동하고 테이블 주문 목록/총액이 0으로 리셋된다.
- Given 이용 완료 후, When 고객이 새 주문을 생성하면, Then 새로운 테이블 세션이 자동으로 시작된다.

### US-A09: 과거 주문 내역 조회
**As a** 관리자,
**I want to** 테이블의 과거 주문 내역을 조회하고 싶다,
**So that** 이전 고객들의 주문 이력을 확인할 수 있다.

**Acceptance Criteria:**
- Given 테이블 카드에서, When 과거 내역 버튼을 클릭하면, Then 해당 테이블의 과거 주문 목록이 시간 역순으로 표시된다.
- Given 과거 내역 화면에서, When 날짜 필터를 적용하면, Then 해당 날짜 범위의 주문만 표시된다.
- Given 각 과거 주문에서, When 상세를 확인하면, Then 주문 번호, 시각, 메뉴 목록, 총 금액, 이용 완료 시각이 표시된다.
- Given 과거 내역 화면에서, When 닫기 버튼을 클릭하면, Then 대시보드로 복귀한다.

## Epic: 메뉴 관리 (FR-09)

### US-A10: 메뉴 조회
**As a** 관리자,
**I want to** 등록된 메뉴를 카테고리별로 조회하고 싶다,
**So that** 현재 메뉴 구성을 파악할 수 있다.

**Acceptance Criteria:**
- Given 메뉴 관리 화면에 접근했을 때, When 화면이 로드되면, Then 카테고리별로 메뉴 목록이 표시된다.

### US-A11: 메뉴 등록
**As a** 관리자,
**I want to** 새로운 메뉴를 등록하고 싶다,
**So that** 고객에게 새 메뉴를 제공할 수 있다.

**Acceptance Criteria:**
- Given 메뉴 등록 화면에서, When 메뉴명, 가격, 설명, 카테고리, 이미지를 입력하고 등록하면, Then 새 메뉴가 생성되고 메뉴 목록에 표시된다.
- Given 필수 필드가 비어있을 때, When 등록을 시도하면, Then 검증 에러 메시지가 표시된다.
- Given 가격이 유효 범위를 벗어날 때, When 등록을 시도하면, Then 가격 범위 에러 메시지가 표시된다.

### US-A12: 메뉴 수정
**As a** 관리자,
**I want to** 기존 메뉴 정보를 수정하고 싶다,
**So that** 가격 변경이나 설명 업데이트를 할 수 있다.

**Acceptance Criteria:**
- Given 메뉴 목록에서, When 메뉴 수정 버튼을 클릭하면, Then 기존 정보가 채워진 수정 폼이 표시된다.
- Given 수정 폼에서, When 정보를 변경하고 저장하면, Then 메뉴 정보가 업데이트되고 목록에 반영된다.

### US-A13: 메뉴 삭제
**As a** 관리자,
**I want to** 메뉴를 삭제하고 싶다,
**So that** 더 이상 제공하지 않는 메뉴를 제거할 수 있다.

**Acceptance Criteria:**
- Given 메뉴 목록에서, When 메뉴 삭제 버튼을 클릭하면, Then 확인 팝업이 표시된다.
- Given 확인 팝업에서, When 삭제를 확정하면, Then 메뉴가 삭제되고 목록에서 제거된다.

### US-A14: 메뉴 노출 순서 조정
**As a** 관리자,
**I want to** 메뉴의 노출 순서를 조정하고 싶다,
**So that** 고객에게 원하는 순서로 메뉴를 보여줄 수 있다.

**Acceptance Criteria:**
- Given 메뉴 관리 화면에서, When 메뉴 순서를 드래그 앤 드롭 또는 순서 변경 버튼으로 조정하면, Then 변경된 순서가 저장되고 고객 화면에 반영된다.
