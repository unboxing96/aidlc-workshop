## Q3 설명: SSE 이벤트 발행 실패 시 주문 처리

### Option A: 주문은 성공 처리 (SSE 실패는 무시)
- **영향**: 주문은 DB에 정상 저장됨. 관리자 대시보드에 실시간 알림이 안 갈 수 있음.
- **장점**: 주문 안정성 보장. 고객은 항상 주문 성공 경험.
- **단점**: 관리자가 새 주문을 놓칠 수 있음 (대시보드 새로고침으로 확인 가능).
- **적합**: 주문 데이터 무결성이 최우선인 경우.

### Option B: 주문도 롤백 (SSE 발행 필수)
- **영향**: SSE 발행 실패 시 주문 자체가 취소됨. 고객에게 에러 표시.
- **장점**: 관리자가 모든 주문을 실시간으로 확인 보장.
- **단점**: SSE 연결 문제로 정상 주문이 실패할 수 있음. 고객 경험 저하.
- **적합**: 실시간 알림이 절대적으로 필요한 경우.
