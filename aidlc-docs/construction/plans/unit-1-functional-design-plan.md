## 명확화 질문

### Q1: 테이블 이용 완료 시 주문 상태 제한
테이블 이용 완료(US-A08) 시, PENDING/PREPARING 상태의 주문이 남아있으면 어떻게 처리할까요?

A) 이용 완료 차단 — 모든 주문이 COMPLETED여야만 이용 완료 가능
B) 강제 완료 — 남은 주문을 자동으로 COMPLETED 처리 후 이용 완료
C) 제한 없음 — 주문 상태와 무관하게 이용 완료 가능 (모든 주문이 히스토리로 이동)

[Answer]: B. 남은 주문은 취소 처리.

### Q2: 테이블 목록 조회 API 접근 권한
GET /api/tables (전체 테이블 목록)는 관리자 전용인가요?

A) 관리자 전용 (JWT 필요)
B) 인증 불필요 (공개)

[Answer]: A

### Q3: 관리자 회원가입 제한
관리자 회원가입(US-A01)에 제한이 있나요?

A) 누구나 가입 가능 (제한 없음)
B) 기존 관리자의 승인 필요
C) 초대 코드 필요

[Answer]: A

### Q4: 테이블 이용 완료 시 고객 화면 처리
이용 완료 후 해당 테이블의 고객 태블릿 화면은 어떻게 되나요?

A) 메뉴 화면 유지 (새 세션으로 자동 전환, 고객은 계속 주문 가능)
B) 빈 화면/안내 메시지 표시 (새 고객 대기 상태)

[Answer]: A

### Q5: 관리자 비밀번호 규칙
비밀번호에 최소 길이나 복잡도 규칙이 필요한가요?

A) 최소 길이만 (예: 4자 이상) — PoC 수준
B) 복잡도 규칙 (대소문자+숫자+특수문자, 8자 이상)
C) 제한 없음

[Answer]: A
